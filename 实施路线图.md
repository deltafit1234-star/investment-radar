# 投资雷达：实施路线图

> 本文档为「投资雷达」产品制定详细的实施计划，包括技术选型、开发任务分解、里程碑排期。
> 适用对象：人民币科技基金 ToB 服务平台

---

## 一、总体实施策略

### 1.1 核心原则

```
快速验证 → 最小闭环 → 迭代优化 → 平台化扩展
    │           │           │           │
    ▼           ▼           ▼           ▼
  第1周       第2-3周     第4-6周     第7-8周
  本地跑通    单赛道验证   多赛道扩展   平台化
```

### 1.2 阶段总览

| 阶段 | 时间 | 目标 | 交付物 |
|------|------|------|--------|
| **Phase 0** | 第1周 | 本地最小闭环跑通 | 端到端可运行的MVP |
| **Phase 1** | 第2-3周 | 单赛道验证 | AI大模型赛道完整跑通 |
| **Phase 2** | 第4-6周 | 多赛道扩展 | 机器人 + 脑机接口上线 |
| **Phase 3** | 第7-8周 | 平台化/多租户 | 多租户架构 + 机构平台接入 |

## 三、详细实施计划

### Phase 0：本地最小闭环（第1周）

**目标**：本地机器上跑通端到端流程，验证可行性

#### 时间分配

```
Day 1 (周一): 项目初始化 + GitHub数据采集
Day 2 (周二): arXiv数据采集 + 变化检测
Day 3 (周三): LLM信号丰富化
Day 4 (周四): 微信推送集成
Day 5 (周五): 端到端测试 + 文档
```

#### Day 1：项目初始化 + GitHub数据采集

**任务**：

```
□ 1.1 创建项目结构
   ├── config/
   ├── src/core/
   ├── src/采集/
   ├── tests/
   └── requirements.txt

□ 1.2 配置虚拟环境和依赖
   python -m venv venv
   pip install -r requirements.txt

□ 1.3 编写 GitHub 数据采集
   ├── GitHubSourceCollector 类
   ├── 采集 Trending Repos
   ├── 采集指定项目的 Star 历史
   └── 存储到 SQLite

□ 1.4 验证
   运行采集脚本，确认数据写入数据库
```

**验收标准**：
```bash
python -c "from src.采集.github import GitHubSourceCollector; c = GitHubSourceCollector(); print(c.fetch_trending())"
# 输出: [{"name": "xxx", "stars": 1234, ...}, ...]
```

#### Day 2：arXiv数据采集 + 变化检测

**任务**：

```
□ 2.1 编写 arXiv 数据采集
   ├── ArxivSourceCollector 类
   ├── 按关键词过滤（AI大模型相关）
   ├── 提取: 标题、摘要、作者、发布日期
   └── 存储到 SQLite

□ 2.2 编写变化检测
   ├── StarSurgeDetector（环比异常）
   ├── PaperBurstDetector（论文激增）
   └── 定义阈值: Star增长 > 30% / 论文 > 10篇/天

□ 2.3 验证
   运行检测，确认异常信号能被识别
```

**验收标准**：
```bash
# 模拟数据测试
python -c "
from src.检测.star_detector import StarSurgeDetector
d = StarSurgeDetector()
result = d.detect({'current': 15000, 'previous': 10000})
print(result)  # {'alert': True, 'priority': 'HIGH', 'growth': 0.5}
"
```

#### Day 3：LLM信号丰富化

**任务**：

```
□ 3.1 集成 OpenAI API
   ├── 配置 .env (OPENAI_API_KEY)
   └── 编写 enricher.py

□ 3.2 信号丰富化逻辑
   输入: 原始信号 {"type": "star_surge", "repo": "llama3", "stars": 12000}
   输出: 丰富后信号 {"type": "star_surge", "repo": "llama3", "stars": 12000,
                     "summary": "Llama3发布后7天Star增长45%，...",
                     "meaning": "开源模型能力逼近GPT-4，应用层机会窗口收窄",
                     "priority": "HIGH"}

□ 3.3 验证
   实际调用 OpenAI API，确认输出符合预期
```

**验收标准**：
```bash
python -c "
from src.分析.enricher import SignalEnricher
e = SignalEnricher()
result = e.enrich({'type': 'star_surge', 'repo': 'llama3', 'stars': 12000})
print(result['summary'])
# 输出: "Llama3发布后7天Star增长45%，主要来自..."
print(result['meaning'])
# 输出: "开源模型能力逼近GPT-4，应用层机会窗口收窄"
"
```

#### Day 4：微信推送集成

**任务**：

```
□ 4.1 设计推送格式
   ├── 每日简报格式
   ├── 实时预警格式
   └── 信号详情格式

□ 4.2 编写推送模块
   ├── NotificationRouter
   ├── WechatNotifier

□ 4.3 集成 Hermes 推送
   ├── 通过 send_message 发送到当前微信对话
```
